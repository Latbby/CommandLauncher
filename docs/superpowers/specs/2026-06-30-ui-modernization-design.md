# 命令启动器界面现代化设计

## 背景

当前项目是基于 Python 和 PySide6 的 Windows 桌面工具。现有界面已经具备项目列表、内置启动命令、全局命令和项目命令管理能力，但布局以纵向堆叠为主，按钮层级不明显，视觉上偏原始。

本次优化目标是让界面更现代、更清晰，同时保持轻量工具属性，不更换技术栈，不引入重型前端框架。

## 目标

- 保留 PySide6 和 Qt Widgets 技术栈，降低改动和打包风险。
- 通过布局重组提升信息层级，减少右侧内容堆叠感。
- 通过 QSS 统一颜色、间距、边框、按钮状态和列表状态。
- 保持启动项目、打开终端、打开资源管理器、管理命令等既有功能不变。
- 保持 Windows 桌面工具的紧凑、高效、低干扰体验。

## 非目标

- 不重写为 Web、Electron 或 Tauri 应用。
- 不引入复杂主题系统或大型第三方 UI 库。
- 不改变配置文件结构和命令执行逻辑。
- 不新增与界面现代化无关的功能。

## 推荐方案

采用“PySide6 + Qt Widgets + QSS + 局部布局重组”的方案。保留现有主窗口和业务逻辑，只调整界面组织方式、控件选择和全局样式。

该方案的收益是改动范围可控，能明显改善观感，同时不会影响当前 PyInstaller 打包链路。

## 界面结构

主窗口继续使用 `QMainWindow`，中心区域改为 `QSplitter`，左右两栏可以拖动宽度。

左侧为项目栏：

- 使用 `QFrame` 作为面板容器。
- 使用 `QLabel` 显示“项目”标题。
- 使用 `QListWidget` 显示项目列表。
- 底部放置“添加”和“移除”两个次级按钮。

右侧为内容区：

- 顶部显示当前项目名称和路径。
- 项目名称使用更高权重字号，路径使用小号灰色文本。
- 中部放置三个主操作按钮：打开 CMD、PowerShell、资源管理器。
- 下部使用 `QTabWidget` 区分“全局命令”和“项目命令”。
- 每个 Tab 内部包含命令列表和“添加 / 编辑 / 删除”按钮组。

## 组件设计

保留组件：

- `QMainWindow`：继续作为主窗口。
- `QListWidget`：继续承载项目和命令列表。
- `QPushButton`：继续承载主要操作。
- `QFrame`：继续作为面板容器。

新增或调整组件：

- `QSplitter`：用于左右栏拖动分隔。
- `QTabWidget`：用于全局命令和项目命令切换。
- `QStatusBar`：用于显示轻量状态提示，例如路径不存在或命令已启动。

按钮分为三类：

- 主按钮：打开 CMD、PowerShell、资源管理器。
- 次级按钮：添加、编辑。
- 危险按钮：移除项目、删除命令。

## 样式设计

全局样式继续集中在 `src/command_launcher/ui/styles.py`。

基础色彩：

- 主背景：浅灰色。
- 面板背景：白色。
- 主色：蓝色，用于主要启动操作和选中状态。
- 文本主色：接近黑色。
- 文本次色：灰色。
- 危险色：红色，用于删除类操作。

视觉规则：

- 面板使用细边框和 8px 圆角。
- 列表项增加固定高度，提供 hover 和 selected 状态。
- 主按钮使用蓝底白字。
- 次级按钮使用白底灰边框。
- 危险按钮使用红色文本或红色 hover 状态。
- Tab 使用轻量分段样式，避免厚重卡片感。

## 交互设计

- 选择项目后，右侧立即刷新项目名称、路径和项目命令。
- 如果项目路径不存在，内置启动按钮禁用，并通过状态栏提示。
- 双击命令列表项继续执行命令，保持原有交互。
- 添加、编辑、删除命令仍使用现有对话框。
- 命令执行失败仍使用弹窗提示，因为该场景需要明确反馈。

## 数据和业务逻辑

本次优化不改变数据模型、配置读写和命令执行逻辑。

涉及模块保持原边界：

- `main_window.py`：负责布局、信号连接和界面刷新。
- `styles.py`：负责全局 QSS 样式。
- `dialogs.py`：继续负责命令编辑对话框。
- `config_store.py`、`models.py`、`command_runner.py` 不做功能性改动。

## 测试和验证

需要执行自动化测试：

- `python -m pytest`

需要手动验证以下用例：

- 入参：启动应用。出参：主窗口正常显示，左右两栏布局正常。
- 入参：添加一个有效项目目录。出参：项目出现在左侧列表，右侧显示项目名称和路径。
- 入参：选择一个存在的项目目录。出参：CMD、PowerShell、资源管理器按钮可用。
- 入参：选择一个路径不存在的项目。出参：内置启动按钮禁用，并显示状态提示。
- 入参：添加全局命令。出参：命令出现在“全局命令”Tab。
- 入参：添加项目命令。出参：命令出现在“项目命令”Tab。
- 入参：双击命令项。出参：命令从当前项目目录启动。

## 风险

- QSS 选择器过宽可能影响对话框控件，需要控制选择器范围。
- `QSplitter` 和 `QTabWidget` 会改变布局结构，需要确保现有测试中引用的属性名保持不变。
- 若后续引入图标库，可能增加打包复杂度；本次设计暂不引入图标依赖。
